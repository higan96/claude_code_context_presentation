# なぜコンテキスト管理が必要か

## 要旨

コンテキストウィンドウの肥大化は、指示の喪失・応答品質の低下・コスト増加を引き起こす。自動圧縮に任せるだけでは「何が消えるか」を制御できないため、能動的な管理が必要。

## 根拠（公式ドキュメントからの抽出）

### コンテキスト肥大化の問題

出典: https://code.claude.com/docs/en/best-practices

コンテキストが大きくなると起こる問題：
- **初期の指示が失われる** — 自動圧縮で詳細な指示が要約・削除される
- **応答品質の低下** — 関連情報がノイズに埋もれる
- **トークンコストの増加** — すべての入出力がトークンとして課金される
- **CLAUDE.mdの遵守率低下** — コンテキストが膨れると指示の遵守率が下がる

### ベストプラクティス: "Manage context aggressively"

出典: https://code.claude.com/docs/en/best-practices

公式が推奨する積極的なコンテキスト管理：
- 2回以上同じ問題で修正を繰り返したら、`/clear` してより詳細なプロンプトで再開
- 無関係なタスクに切り替える際は `/clear` を使用
- 長いセッションでは定期的に `/compact` を実行
- 永続的なルールは会話ではなく CLAUDE.md に記述

### MCP サーバーのコンテキストオーバーヘッド

出典: https://code.claude.com/docs/en/costs

- 接続中のMCPサーバーはツール定義を毎リクエストで追加
- 複数サーバーでかなりのコンテキストを消費する可能性
- `/mcp` でサーバーごとのコストを確認し、不要なサーバーは切断すべき

### CLAUDE.md のサイズと遵守率の関係

出典: https://code.claude.com/docs/en/memory

- CLAUDE.mdは200行以下を推奨
- 長いファイルはトークン消費量が増えて遵守率が低下
- 大きな指示は `.claude/rules/` に分割するか、スキルに移動

## スライドで使う主張候補

1. **コンテキスト肥大化 = 指示忘れ** — 「言ったはずなのにやってくれない」の原因はコンテキスト圧縮
2. **自動圧縮は保険であって戦略ではない** — 何が消えるかを選べない
3. **コスト直結** — 無駄なコンテキスト = 無駄な課金
4. **公式が "aggressively" と言っている** — 受動的な管理では不十分
